Tests automatisés

Ce stage de six semaines a consisté en la mise en place d'une librairie Playwright pour permettre l'automatisation de robot de tests automatisé sur des applications WEB de la société CNP Assurances.

Description

Mon stage s'est déroulé au sein de CNP Assurances à Angers. Je me suis trouvé dans l'équipe Tests automatisés tout au long de ce stage.

Le sujet principal du stage a consisté à mettre en place une librairie Playwright dans le socle automatisation écrit en Robot Framework afin de remplacer l’actuelle librairie Selenium. La CNP a souhaité passer à cette nouvelle librairie car certains robots ne fonctionnaient pas totalement avec la librairie Selenium notamment quand les applications étaient amenées à utiliser des pop-ups. De ce fait, les applications ne pouvaient pas être testées en mode automatisé ce qui obligeait à faire des tests manuels. Ces tests manuels pouvaient prendre du temps. Il faut noter que la nouvelle librairie mise en place ne remplace pas totalement l’ancienne. Elle permet simplement d’avoir une deuxième option si nécessaire.

Un second objectif a été de renforcer les tests unitaires existants pour la librairie Selenium et les adapter si nécessaire à la librairie Playwright.

Le dernier objectif était de mettre en place un socle automatisation avec la librairie Appium pour permettre des tests automatisés pour mobile.

Travaux réalisés

Les travaux principaux réalisés durant ce stage ont été les suivants.

1. Réalisation du premier robot de test

J'ai d'abord commencé par automatiser un site WEB pour voir le fonctionnement de Robot Framework et des librairies Selenium et Playwright.

2. Création du socle Playwright

Ensuite, j'ai procédé à la création du socle Playwright en utilisant le socle Selenium déjà présent et en reproduisant le comportement attendu. Toutes les fonctions existantes pour Selenium ont été reproduites avec la librairie Playwright.

3. Création d'un robot de test sur une application CNP

Une équipe de la CNP a fait une demande de création de robot pour automatiser un scénario de test. Ce robot fait donc un parcours précis en fonction d'un jeux de données et remplit les champs que le testeur souhaite alimenter.

4. Renforcement de tests unitaires

J'ai repris une suite de test unitaire existante qui possédait 88 tests. J'ai cherché de nouveaux tests pertinents à ajouter pour sécuriser la validité des socles Playwright et Selenium. Je suis passé de 88 à 180 TUs.

5. Création du socle Appium

Ma dernière tâche a été d'initialiser un socle Appium pour permettre de faire des tests automatisés sur mobile. Cette initialisation permet déjà de faire des tests basiques sur mobile.

Compétences aquises

Ce stage m'a permis d'apprendre à utiliser de nouveaux outils en lisant de la documentation en ligne. Il m'a aussi appris à automatiser des tests à l'aide de Robot Framework. Les développements effectués m'ont permis de découvrir la méthodologie Test Driven Development.
